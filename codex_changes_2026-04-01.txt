Hestia-PHP Change Log
Date: 2026-04-01
Prepared by: Codex

Summary
This document records the class cleanup and fatal-error prevention work completed across the normal classes and admin classes.

Primary Goal
- Stop fatal errors caused by pages calling methods that do not exist on the class instance being used.
- Remove redundant methods spread across classes when another class is the more appropriate owner.
- Update all known usages so each call points to the correct class and method.

1. Fatal Error Fix Completed
Original fatal:
- Call to undefined method Property::get_landlord_property_count()
- Source file: landlord/landlord-profile.php

What was done:
- Added the missing landlord-facing methods to classes/Property.php:
  - get_landlord_property_count()
  - get_landlord_dashboard_stats()
  - get_landlord_applications()
- Hardened get_landlord_properties() by safely casting limit and offset values.
- Ensured these methods return safe fallback values on database failure:
  - 0 for counts
  - [] for record lists

Reason:
- landlord/landlord-profile.php and landlord/my-properties.php were using Property methods that did not exist.

2. Property Class Consolidation
Canonical owner chosen:
- classes/Property.php is now the main owner for property-specific read helpers.

Methods kept or added in classes/Property.php:
- get_property_by_id()
- get_landlord_properties()
- get_landlord_property_count()
- get_landlord_dashboard_stats()
- get_landlord_applications()
- get_property_images()
- get_property_amenities()

Methods removed from classes/Property.php:
- get_property_types()
  - Reason: property types belong more naturally to the State/location/support lookup flow already used elsewhere.
- legacy/redundant helper methods that were not part of the cleaned call flow:
  - get_landlord_prop()
  - get_amenities_by_property()
  - get_images()
  - get_stats()
  - apply_for_property()
  - is_saved()

Reason:
- These methods were either duplicated elsewhere, superseded by better-named methods, or belonged to a different domain class.

3. Admin Class Consolidation
Canonical owner chosen:
- admin/classes/Admin.php should handle admin-only concerns, not duplicate property/user domain methods already owned elsewhere.

Methods removed from admin/classes/Admin.php:
- get_user_by_id()
- update_user_status()
- get_user_active_status()
- get_landlord_properties()
- get_landlord_dashboard_stats()
- get_property_by_id()
- get_property_images()
- get_property_amenities()
- get_landlord_details()
- toggle_status()

Reason:
- These methods duplicated functionality already provided by:
  - classes/User.php
  - classes/Property.php
- Keeping duplicate query logic in Admin created drift risk and was a direct path to future fatal errors.

Methods still kept in admin/classes/Admin.php:
- admin authentication
- admin dashboard totals/stats
- admin search methods
- recent dashboard data
- property moderation/review actions
- admin profile update
- admin logout

4. User Domain Usage Alignment
Canonical owner chosen:
- classes/User.php is the source of truth for user fetch and user status changes.

Methods now used for user data/status:
- get_user_by()
- set_status()

Files updated to use User instead of Admin duplicates:
- admin/views/landlord-details.php
- admin/process/process_get-tenant.php
- admin/process/process_toggle.php

5. State and Wishlist Domain Alignment
State class:
- classes/State.php remains the owner of get_property_types()
- views/properties.php was updated to use State::get_property_types() instead of Property::get_property_types()

Wishlist class:
- Wishlist responsibility was kept in classes/Wishlist.php
- views/properties.php was updated so wishlist state is derived from Wishlist data instead of using the removed Property::is_saved()
- Wishlist IDs are now loaded once before the property loop for better efficiency

6. Call Site Updates
The following files were updated so they now call the correct owning class:

Updated admin files
- admin/views/landlord-details.php
  - Switched landlord lookup to User::get_user_by('id', ...)
  - Switched landlord properties/stats lookup to Property methods

- admin/property-review.php
  - Switched property details, images, and amenities to Property methods

- admin/process/process_get-property.php
  - Switched property lookup to Property::get_property_by_id()

- admin/process/process_get-tenant.php
  - Switched tenant lookup to User::get_user_by('id', ...)

- admin/process/process_toggle.php
  - Property toggle now reads property info from Property
  - User toggle now reads user info from User
  - User status update now uses User::set_status()

Updated public/normal files
- landlord/landlord-profile.php
  - Now supported by actual methods that exist in Property

- landlord/my-properties.php
  - Now supported by actual methods that exist in Property

- views/property-details.php
  - Switched from removed legacy methods:
    - get_images()
    - get_amenities_by_property()
  - Now uses:
    - get_property_images()
    - get_property_amenities()

- views/properties.php
  - Switched property type loading to State::get_property_types()
  - Replaced removed Property::is_saved() usage with Wishlist-driven saved ID lookup

7. Safety Improvements
- Added try/catch handling around get_property_by_id() in classes/Property.php so it fails safely.
- Standardized several property data methods to return stable defaults instead of causing downstream breakage.
- Reduced the chance of “method exists in one class copy but not another” regressions by removing redundant admin-side duplicates.

9. Remaining Notes

- Functional browser testing was not performed in this change log pass. Recommended manual checks:
  - landlord dashboard
  - landlord my-properties page

10. Outcome
The codebase is now more consistent about method ownership:
- Property data comes from Property
- User data/status comes from User
- Property type lookup comes from State
- Wishlist state comes from Wishlist
- Admin keeps admin-only responsibilities

This reduces duplication, lowers drift between admin and non-admin code paths, and helps prevent the kind of undefined-method fatal error that originally occurred.
