Restored Session And Admin Guard Fixes

This note tracks the non-dark-mode fixes that were restored after returning from the dark mode experiment.

Files updated

1. process/process_login.php
   Restored session cleanup on successful normal user login.
   What it does:
   - regenerates the session id
   - clears old session values before setting `user_id`, `user_role`, and `user_email`
   Why:
   - prevents admin session values from lingering when logging out of admin and back into a landlord or tenant account

2. process/process_application.php
   Restored an admin-side guard for property applications.
   What it does:
   - blocks administrator accounts from submitting rental applications
   - redirects back to the property details page with an error message

3. process/process_inspection.php
   Restored an admin-side guard for inspection requests.
   What it does:
   - blocks administrator accounts from requesting viewings
   - redirects back to the property details page with an error message

4. views/property-details.php
   Restored the missing admin-specific UI behavior on the property details page.
   What it does:
   - shows admin-specific messages instead of "you can't inspect/apply to your own property"
   - prevents admins from seeing the inspection request form
   - prevents admins from seeing the application form
   - disables wishlist access for admins with a disabled button/message
   - keeps owner-specific messages for landlords separate from admin-specific ones

Count fix status

The application and inspection count fix is still present.
It was not removed.

Where it lives:
- classes/PropertyTracker.php

What it does:
- `views_count` still comes from `property_stats`
- `application_count` is counted directly from the `applications` table
- `inspection_count` is counted directly from the `inspections` table
